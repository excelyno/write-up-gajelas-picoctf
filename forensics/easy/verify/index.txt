1️⃣ Struktur dasarnya: for ... do ... done

Bagian ini melakukan iterasi (loop) terhadap semua entri di dalam folder files:

for f in files/*;


files/* = wildcard (glob) yang mencocokkan semua file di dalam direktori files.

Setiap hasil dicatat ke variabel f.

Loop berjalan file demi file.

Contoh:

files/a1
files/b2
files/c3


Loop akan berjalan 3 kali dengan nilai:

f=files/a1

f=files/b2

f=files/c3

2️⃣ Menjalankan dekripsi untuk setiap file
./decrypt.sh 


Ini memanggil script dekripsi dengan parameter file current ().

Script decrypt.sh:

memverifikasi file valid

mencoba mendekripsi file

mencetak flag jika file benar

mencetak error jika gagal

3️⃣ Menghilangkan error agar output bersih
2>/dev/null


Artinya:

Semua error (stderr) diarahkan ke /dev/null

Dengan demikian, hanya output sukses yang terlihat.

Ini penting karena decrypt.sh mencetak:

Error: Failed to decrypt ...


Jika tidak dibuang, grep akan kebanjiran noise.

4️⃣ Menyaring hanya baris yang mengandung kata pico
| grep -i pico


Pipeline ini bekerja seperti berikut:

Output hasil dekripsi dikirim ke grep.

grep -i = pencarian case-insensitive.

Hanya menampilkan baris yang mengandung kata:

pico


Dalam CTF picoCTF, flag selalu berbentuk:

picoCTF{...}


Jadi kita hanya fokus pada flag.

Jika file tidak berisi flag, grep menghasilkan exit code gagal.

5️⃣ Operator logika untuk menentukan file yang benar
&& echo  


&& artinya:

“Jalankan perintah kedua hanya jika perintah sebelumnya berhasil.”

Berarti:

Jika grep menemukan flag → echo dijalankan.

Jika tidak ditemukan → echo dilewati.
